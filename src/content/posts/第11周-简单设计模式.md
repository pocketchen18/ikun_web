## 一、什么是设计模式？

### 1.1 概念

设计模式是**解决常见问题的最佳实践**，就像做菜的经典菜谱。

```
问题：如何保证一个类只有一个实例？
方案：单例模式

问题：如何创建不同类型的对象？
方案：工厂模式
```

### 1.2 设计模式分类

| 类型 | 说明 | 包含模式 |
|------|------|---------|
| 创建型 | 如何创建对象 | 单例、工厂、建造者 |
| 结构型 | 如何组合类 | 装饰、适配器、代理 |
| 行为型 | 如何交互 | 观察者、策略、MVC |

## 二、单例模式（Singleton）

### 2.1 概念

**保证一个类只有一个实例，并提供全局访问点。**

**现实比喻：** 一个国家只有一个总统

```
总统办公室
    │
    ── 总统（只有一个）
         ├── 处理事务A
         ├── 处理事务B
         └── 处理事务C
```

### 2.2 饿汉式单例

程序启动时就创建实例。

```cpp
#include <iostream>

class Singleton {
private:
    // 私有构造函数，防止外部new
    Singleton() {
        std::cout << "单例创建" << std::endl;
    }
    
    // 静态实例
    static Singleton* instance;
    
public:
    // 禁止拷贝和赋值
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;
    
    // 获取实例
    static Singleton* getInstance() {
        return instance;
    }
    
    void doSomething() {
        std::cout << "单例在工作..." << std::endl;
    }
};

// 类外初始化静态成员
Singleton* Singleton::instance = new Singleton();

int main() {
    // 获取单例
    Singleton* s1 = Singleton::getInstance();
    Singleton* s2 = Singleton::getInstance();
    
    // 验证是同一个实例
    std::cout << "s1地址: " << s1 << std::endl;
    std::cout << "s2地址: " << s2 << std::endl;
    std::cout << "是同一个实例: " << (s1 == s2) << std::endl;
    
    s1->doSomething();
    
    return 0;
}
```

### 2.3 懒汉式单例（线程安全）

第一次使用时才创建实例。

```cpp
#include <iostream>
#include <mutex>

class LazySingleton {
private:
    LazySingleton() {
        std::cout << "懒汉单例创建" << std::endl;
    }
    
    static LazySingleton* instance;
    static std::mutex mtx;
    
public:
    LazySingleton(const LazySingleton&) = delete;
    LazySingleton& operator=(const LazySingleton&) = delete;
    
    static LazySingleton* getInstance() {
        // 双重检查锁定
        if (instance == nullptr) {
            std::lock_guard<std::mutex> lock(mtx);
            if (instance == nullptr) {
                instance = new LazySingleton();
            }
        }
        return instance;
    }
    
    void doSomething() {
        std::cout << "懒汉单例在工作..." << std::endl;
    }
};

LazySingleton* LazySingleton::instance = nullptr;
std::mutex LazySingleton::mtx;

int main() {
    LazySingleton* s = LazySingleton::getInstance();
    s->doSomething();
    return 0;
}
```

### 2.4 C++11 最简单的单例

```cpp
#include <iostream>

class ModernSingleton {
private:
    ModernSingleton() {}
    
public:
    ModernSingleton(const ModernSingleton&) = delete;
    ModernSingleton& operator=(const ModernSingleton&) = delete;
    
    // C++11保证局部静态变量只初始化一次，且线程安全
    static ModernSingleton& getInstance() {
        static ModernSingleton instance;
        return instance;
    }
    
    void doSomething() {
        std::cout << "现代单例在工作..." << std::endl;
    }
};

int main() {
    ModernSingleton::getInstance().doSomething();
    return 0;
}
```

- 配置管理器（全局唯一配置）
- 日志管理器（全局唯一日志）
- 数据库连接池
- 线程池

## 三、工厂模式（Factory）

### 3.1 简单工厂

**概念：** 用一个工厂类来创建不同类型的对象。

**现实比喻：** 汽车工厂

```
汽车工厂
    ├── 生产轿车
    ├── 生产SUV
    └── 生产卡车
```

```cpp
#include <iostream>
#include <string>
#include <memory>

// 产品基类
class Car {
public:
    virtual void drive() = 0;
    virtual ~Car() = default;
};

// 具体产品
class Sedan : public Car {
public:
    void drive() override {
        std::cout << "驾驶轿车..." << std::endl;
    }
};

class SUV : public Car {
public:
    void drive() override {
        std::cout << "驾驶SUV..." << std::endl;
    }
};

class Truck : public Car {
public:
    void drive() override {
        std::cout << "驾驶卡车..." << std::endl;
    }
};

// 简单工厂
class CarFactory {
public:
    static std::unique_ptr<Car> createCar(const std::string& type) {
        if (type == "sedan") {
            return std::make_unique<Sedan>();
        } else if (type == "suv") {
            return std::make_unique<SUV>();
        } else if (type == "truck") {
            return std::make_unique<Truck>();
        }
        return nullptr;
    }
};

int main() {
    auto car1 = CarFactory::createCar("sedan");
    car1->drive();
    
    auto car2 = CarFactory::createCar("suv");
    car2->drive();
    
    return 0;
}
```

### 3.2 工厂方法模式

**概念：** 每个产品有一个对应的工厂。

```cpp
#include <iostream>
#include <memory>

// 产品接口
class Button {
public:
    virtual void render() = 0;
    virtual ~Button() = default;
};

class WindowsButton : public Button {
public:
    void render() override {
        std::cout << "渲染Windows风格按钮" << std::endl;
    }
};

class MacButton : public Button {
public:
    void render() override {
        std::cout << "渲染Mac风格按钮" << std::endl;
    }
};

// 工厂接口
class ButtonFactory {
public:
    virtual std::unique_ptr<Button> createButton() = 0;
    virtual ~ButtonFactory() = default;
};

class WindowsButtonFactory : public ButtonFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<WindowsButton>();
    }
};

class MacButtonFactory : public ButtonFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<MacButton>();
    }
};

int main() {
    // 根据操作系统选择工厂
    std::unique_ptr<ButtonFactory> factory;
    
    #ifdef _WIN32
        factory = std::make_unique<WindowsButtonFactory>();
    #else
        factory = std::make_unique<MacButtonFactory>();
    #endif
    
    auto button = factory->createButton();
    button->render();
    
    return 0;
}
```

### 3.3 抽象工厂模式

**概念：** 创建一系列相关产品的工厂。

```cpp
#include <iostream>
#include <memory>
#include <string>

// 产品族：按钮
class Button {
public:
    virtual void paint() = 0;
    virtual ~Button() = default;
};

class WinButton : public Button {
public:
    void paint() override {
        std::cout << "绘制Windows按钮" << std::endl;
    }
};

class MacButton : public Button {
public:
    void paint() override {
        std::cout << "绘制Mac按钮" << std::endl;
    }
};

// 产品族：复选框
class Checkbox {
public:
    virtual void paint() = 0;
    virtual ~Checkbox() = default;
};

class WinCheckbox : public Checkbox {
public:
    void paint() override {
        std::cout << "绘制Windows复选框" << std::endl;
    }
};

class MacCheckbox : public Checkbox {
public:
    void paint() override {
        std::cout << "绘制Mac复选框" << std::endl;
    }
};

// 抽象工厂
class GUIFactory {
public:
    virtual std::unique_ptr<Button> createButton() = 0;
    virtual std::unique_ptr<Checkbox> createCheckbox() = 0;
    virtual ~GUIFactory() = default;
};

class WinFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<WinButton>();
    }
    std::unique_ptr<Checkbox> createCheckbox() override {
        return std::make_unique<WinCheckbox>();
    }
};

class MacFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<MacButton>();
    }
    std::unique_ptr<Checkbox> createCheckbox() override {
        return std::make_unique<MacCheckbox>();
    }
};

// 客户端代码
class Application {
private:
    std::unique_ptr<Button> button;
    std::unique_ptr<Checkbox> checkbox;
    
public:
    Application(std::unique_ptr<GUIFactory> factory) {
        button = factory->createButton();
        checkbox = factory->createCheckbox();
    }
    
    void paint() {
        button->paint();
        checkbox->paint();
    }
};

int main() {
    // 创建Windows风格界面
    auto winFactory = std::make_unique<WinFactory>();
    Application app(std::move(winFactory));
    app.paint();
    
    return 0;
}
```

- UI组件（不同主题）

## 四、MVC模式

### 4.1 概念

**MVC = Model（模型）+ View（视图）+ Controller（控制器）**

```
┌─────────────────────────────────────────┐
│              MVC 架构                    │
│                                         │
│   ┌─────────┐    更新    ┌─────────┐    │
│   │  Model  │──────────→│  View   │    │
│   │ (数据)  │            │ (界面)  │    │
│   └────┬────┘            └─────────┘    │
│        │                                 │
│        │ 操作                            │
│        ↓                                 │
│   ┌─────────┐    请求    ┌─────────┐    │
│   │Controller│←──────────│  用户   │    │
│   │ (逻辑)  │            │ (输入)  │    │
│   └─────────┘            └─────────┘    │
└─────────────────────────────────────────
```

**各部分职责：**
- **Model（模型）：** 数据和业务逻辑
- **View（视图）：** 用户界面显示
- **Controller（控制器）：** 处理用户输入，协调Model和View

### 4.2 MVC代码示例

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <functional>

// ===== Model（模型）=====
// 负责数据和业务逻辑
class StudentModel {
private:
    int id;
    std::string name;
    std::string grade;
    
public:
    StudentModel(int id, const std::string& name, const std::string& grade)
        : id(id), name(name), grade(grade) {}
    
    int getId() const { return id; }
    std::string getName() const { return name; }
    std::string getGrade() const { return grade; }
    
    void setName(const std::string& name) { this->name = name; }
    void setGrade(const std::string& grade) { this->grade = grade; }
};

// ===== View（视图）=====
// 负责显示
class StudentView {
public:
    void displayStudent(const StudentModel& student) {
        std::cout << "=== 学生信息 ===" << std::endl;
        std::cout << "学号: " << student.getId() << std::endl;
        std::cout << "姓名: " << student.getName() << std::endl;
        std::cout << "班级: " << student.getGrade() << std::endl;
        std::cout << "================" << std::endl;
    }
};

// ===== Controller（控制器）=====
// 负责协调Model和View
class StudentController {
private:
    StudentModel model;
    StudentView view;
    
public:
    StudentController(int id, const std::string& name, const std::string& grade)
        : model(id, name, grade) {}
    
    // 获取数据
    int getStudentId() { return model.getId(); }
    std::string getStudentName() { return model.getName(); }
    std::string getStudentGrade() { return model.getGrade(); }
    
    // 设置数据
    void setStudentName(const std::string& name) {
        model.setName(name);
    }
    
    void setStudentGrade(const std::string& grade) {
        model.setGrade(grade);
    }
    
    // 显示视图
    void updateView() {
        view.displayStudent(model);
    }
};

int main() {
    // 创建控制器（内部包含Model和View）
    StudentController controller(1, "张三", "三年级一班");
    
    // 显示初始数据
    controller.updateView();
    
    // 修改数据
    controller.setStudentName("李四");
    controller.setStudentGrade("四年级二班");
    
    // 更新显示
    controller.updateView();
    
    return 0;
}
```

### 4.3 MVC在Qt中的应用

Qt本身就是基于MVC思想的：

```cpp
// Qt中的MVC对应
Model  → QAbstractItemModel, QSqlTableModel
View   → QTableView, QListView

model->setTable("students");
model->select();

QTableView* view = new QTableView();

```

- Web框架（Django, Spring MVC）
- 桌面应用（Qt, JavaFX）
- 移动应用（iOS, Android）
- 游戏开发

## 五、装饰模式（Decorator）

### 5.1 概念

**动态地给对象添加功能，比继承更灵活。**

**现实比喻：** 给手机加配件

```
裸手机
    ├── 加手机壳
    ├── 加钢化膜
    ── 加支架
    
最终：手机 + 手机壳 + 钢化膜 + 支架
```

```
        ┌──────────────┐
        │   Component  │  ← 组件接口
        └──────┬───────┘
               │
    ┌──────────┼──────────┐
    │          │          │
┌──────┐  ┌──┴───┐  ┌───┴───┐
│Concrete│  │Decorator│  │Decorator│
│Component│  │ (基类)  │  │ (具体)  │
└───────┘  └───┬───┘  └───┬───┘
               │          │
               │     ┌────────┐
               │     │Concrete │
               │     │Decorator│
               │     └─────────┘
```

### 5.2 装饰模式代码示例

```cpp
#include <iostream>
#include <string>
#include <memory>

// 组件接口
class Coffee {
public:
    virtual std::string getDescription() = 0;
    virtual double cost() = 0;
    virtual ~Coffee() = default;
};

// 具体组件：基础咖啡
class SimpleCoffee : public Coffee {
public:
    std::string getDescription() override {
        return "基础咖啡";
    }
    
    double cost() override {
        return 10.0;
    }
};

// 装饰器基类
class CoffeeDecorator : public Coffee {
protected:
    std::unique_ptr<Coffee> decoratedCoffee;
    
public:
    CoffeeDecorator(std::unique_ptr<Coffee> coffee)
        : decoratedCoffee(std::move(coffee)) {}
    
    std::string getDescription() override {
        return decoratedCoffee->getDescription();
    }
    
    double cost() override {
        return decoratedCoffee->cost();
    }
};

// 具体装饰器：加牛奶
class MilkDecorator : public CoffeeDecorator {
public:
    MilkDecorator(std::unique_ptr<Coffee> coffee)
        : CoffeeDecorator(std::move(coffee)) {}
    
    std::string getDescription() override {
        return decoratedCoffee->getDescription() + " + 牛奶";
    }
    
    double cost() override {
        return decoratedCoffee->cost() + 3.0;
    }
};

// 具体装饰器：加糖
class SugarDecorator : public CoffeeDecorator {
public:
    SugarDecorator(std::unique_ptr<Coffee> coffee)
        : CoffeeDecorator(std::move(coffee)) {}
    
    std::string getDescription() override {
        return decoratedCoffee->getDescription() + " + 糖";
    }
    
    double cost() override {
        return decoratedCoffee->cost() + 1.0;
    }
};

// 具体装饰器：加珍珠
class PearlDecorator : public CoffeeDecorator {
public:
    PearlDecorator(std::unique_ptr<Coffee> coffee)
        : CoffeeDecorator(std::move(coffee)) {}
    
    std::string getDescription() override {
        return decoratedCoffee->getDescription() + " + 珍珠";
    }
    
    double cost() override {
        return decoratedCoffee->cost() + 5.0;
    }
};

int main() {
    // 基础咖啡
    auto coffee = std::make_unique<SimpleCoffee>();
    std::cout << coffee->getDescription() << " ￥" << coffee->cost() << std::endl;
    
    // 加牛奶
    coffee = std::make_unique<MilkDecorator>(std::move(coffee));
    std::cout << coffee->getDescription() << " ￥" << coffee->cost() << std::endl;
    
    // 加糖
    coffee = std::make_unique<SugarDecorator>(std::move(coffee));
    std::cout << coffee->getDescription() << " ￥" << coffee->cost() << std::endl;
    
    // 加珍珠
    coffee = std::make_unique<PearlDecorator>(std::move(coffee));
    std::cout << coffee->getDescription() << " ￥" << coffee->cost() << std::endl;
    
    return 0;
}
```

**输出：**
```
基础咖啡 ￥10
基础咖啡 + 牛奶 ￥13
基础咖啡 + 牛奶 + 糖 ￥14
基础咖啡 + 牛奶 + 糖 + 珍珠 ￥19
```

### 5.3 装饰模式应用场景

- Qt样式（QSS装饰控件）
- 日志增强（加时间戳、加级别）
- 网络请求（加认证、加日志）

## 六、建造者模式（Builder）

### 6.1 概念

**分步骤构建复杂对象，让构建过程和表示分离。**

**现实比喻：** 组装电脑

```
组装电脑步骤：
1. 安装CPU
2. 安装内存
3. 安装硬盘
4. 安装显卡
5. 安装电源

不同配置 = 不同步骤组合
```

### 6.2 建造者模式代码示例

```cpp
#include <iostream>
#include <string>
#include <memory>

// 产品：电脑
class Computer {
private:
    std::string cpu;
    std::string ram;
    std::string storage;
    std::string gpu;
    std::string power;
    
public:
    void setCpu(const std::string& cpu) { this->cpu = cpu; }
    void setRam(const std::string& ram) { this->ram = ram; }
    void setStorage(const std::string& storage) { this->storage = storage; }
    void setGpu(const std::string& gpu) { this->gpu = gpu; }
    void setPower(const std::string& power) { this->power = power; }
    
    void showConfig() {
        std::cout << "=== 电脑配置 ===" << std::endl;
        std::cout << "CPU: " << cpu << std::endl;
        std::cout << "内存: " << ram << std::endl;
        std::cout << "硬盘: " << storage << std::endl;
        std::cout << "显卡: " << gpu << std::endl;
        std::cout << "电源: " << power << std::endl;
        std::cout << "================" << std::endl;
    }
};

// 抽象建造者
class ComputerBuilder {
public:
    virtual void buildCpu() = 0;
    virtual void buildRam() = 0;
    virtual void buildStorage() = 0;
    virtual void buildGpu() = 0;
    virtual void buildPower() = 0;
    virtual Computer* getResult() = 0;
    virtual ~ComputerBuilder() = default;
};

// 具体建造者：游戏电脑
class GamingComputerBuilder : public ComputerBuilder {
private:
    Computer* computer;
    
public:
    GamingComputerBuilder() {
        computer = new Computer();
    }
    
    void buildCpu() override {
        computer->setCpu("Intel i9-13900K");
    }
    
    void buildRam() override {
        computer->setRam("32GB DDR5");
    }
    
    void buildStorage() override {
        computer->setStorage("2TB NVMe SSD");
    }
    
    void buildGpu() override {
        computer->setGpu("RTX 4090");
    }
    
    void buildPower() override {
        computer->setPower("1000W 金牌");
    }
    
    Computer* getResult() override {
        return computer;
    }
    
    ~GamingComputerBuilder() override {
        delete computer;
    }
};

// 具体建造者：办公电脑
class OfficeComputerBuilder : public ComputerBuilder {
private:
    Computer* computer;
    
public:
    OfficeComputerBuilder() {
        computer = new Computer();
    }
    
    void buildCpu() override {
        computer->setCpu("Intel i5-13400");
    }
    
    void buildRam() override {
        computer->setRam("16GB DDR4");
    }
    
    void buildStorage() override {
        computer->setStorage("512GB SSD");
    }
    
    void buildGpu() override {
        computer->setGpu("集成显卡");
    }
    
    void buildPower() override {
        computer->setPower("500W");
    }
    
    Computer* getResult() override {
        return computer;
    }
    
    ~OfficeComputerBuilder() override {
        delete computer;
    }
};

// 指挥者：控制建造流程
class Director {
public:
    Computer* buildComputer(ComputerBuilder& builder) {
        builder.buildCpu();
        builder.buildRam();
        builder.buildStorage();
        builder.buildGpu();
        builder.buildPower();
        return builder.getResult();
    }
};

int main() {
    Director director;
    
    // 建造游戏电脑
    GamingComputerBuilder gamingBuilder;
    Computer* gamingPC = director.buildComputer(gamingBuilder);
    std::cout << "游戏电脑:" << std::endl;
    gamingPC->showConfig();
    
    // 建造办公电脑
    OfficeComputerBuilder officeBuilder;
    Computer* officePC = director.buildComputer(officeBuilder);
    std::cout << "办公电脑:" << std::endl;
```

### 7.4 观察者模式（信号槽机制）

Qt的信号槽就是观察者模式的完美实现！

```cpp
#include <QObject>
#include <QDebug>

// 被观察者（主题）
class DataModel : public QObject {
    Q_OBJECT
private:
    int value_;
    
public:
    int value() const { return value_; }
    
    void setValue(int newValue) {
        if (value_ != newValue) {
            value_ = newValue;
            emit valueChanged(value_);  // 通知所有观察者
        }
    }
    
signals:
    void valueChanged(int newValue);
};

// 观察者
class DataObserver : public QObject {
    Q_OBJECT
public slots:
    void onValueChanged(int newValue) {
        qDebug() << "数据已更新:" << newValue;
        // 更新UI、保存数据等
    }
};

// 使用
void setupObserver() {
    DataModel* model = new DataModel;
    DataObserver* observer = new DataObserver;
    
    // 注册观察者
    connect(model, &DataModel::valueChanged, 
            observer, &DataObserver::onValueChanged);
    
    // 修改数据，观察者自动收到通知
    model->setValue(100);
    model->setValue(200);
}
```

### 7.7 MVC模式在Qt中的深度应用

#### Qt Model/View架构

```cpp
#include <QAbstractTableModel>
#include <QTableView>
#include <QVector>
#include <QVariant>

// Model（模型）
class StudentModel : public QAbstractTableModel {
private:
    struct Student {
        int id;
        QString name;
        int age;
        QString grade;
    };
    
    QVector<Student> students_;
    QStringList headers_ = {"学号", "姓名", "年龄", "班级"};
    
public:
    int rowCount(const QModelIndex& parent = QModelIndex()) const override {
        return students_.size();
    }
    
    int columnCount(const QModelIndex& parent = QModelIndex()) const override {
        return 4;
    }
    
    QVariant data(const QModelIndex& index, int role = Qt::DisplayRole) const override {
        if (!index.isValid() || role != Qt::DisplayRole) return QVariant();
        
        const Student& s = students_[index.row()];
        switch (index.column()) {
            case 0: return s.id;
            case 1: return s.name;
            case 2: return s.age;
            case 3: return s.grade;
            default: return QVariant();
        }
    }
    
    QVariant headerData(int section, Qt::Orientation orientation, int role = Qt::DisplayRole) const override {
        if (role == Qt::DisplayRole && orientation == Qt::Horizontal) {
            return headers_[section];
        }
        return QVariant();
    }
    
    // 添加数据
    void addStudent(int id, const QString& name, int age, const QString& grade) {
        int row = students_.size();
        beginInsertRows(QModelIndex(), row, row);
        students_.append({id, name, age, grade});
        endInsertRows();
    }
    
    // 删除数据
    void removeStudent(int row) {
        if (row >= 0 && row < students_.size()) {
            beginRemoveRows(QModelIndex(), row, row);
            students_.removeAt(row);
            endRemoveRows();
        }
    }
    
    // 修改数据
    bool setData(const QModelIndex& index, const QVariant& value, int role = Qt::EditRole) override {
        if (index.isValid() && role == Qt::EditRole) {
            Student& s = students_[index.row()];
            switch (index.column()) {
                case 0: s.id = value.toInt(); break;
                case 1: s.name = value.toString(); break;
                case 2: s.age = value.toInt(); break;
                case 3: s.grade = value.toString(); break;
            }
            emit dataChanged(index, index);
            return true;
        }
        return false;
    }
    
    Qt::ItemFlags flags(const QModelIndex& index) const override {
        return Qt::ItemIsEditable | Qt::ItemIsEnabled | Qt::ItemIsSelectable;
    }
};

// 使用
void setupTableView() {
    // 创建Model
    StudentModel* model = new StudentModel;
    model->addStudent(1, "张三", 20, "计算机1班");
    model->addStudent(2, "李四", 21, "计算机2班");
    model->addStudent(3, "王五", 19, "计算机1班");
    
    // 创建View
    QTableView* view = new QTableView;
    view->setModel(model);  // View绑定Model
    view->show();
}
```

```cpp
#include <QObject>

// 插件接口
class PluginInterface {
public:
    virtual ~PluginInterface() = default;
    virtual QString name() const = 0;
    virtual void execute() = 0;
};

// 具体插件
class ImagePlugin : public PluginInterface {
public:
    QString name() const override { return "图像处理插件"; }
    void execute() override { qDebug() << "执行图像处理"; }
};

class AudioPlugin : public PluginInterface {
public:
    QString name() const override { return "音频处理插件"; }
    void execute() override { qDebug() << "执行音频处理"; }
};

// 插件工厂
class PluginFactory {
private:
    static QMap<QString, PluginInterface*>& registry() {
        static QMap<QString, PluginInterface*> map;
        return map;
    }
    
public:
    // 注册插件
    static void registerPlugin(const QString& type, PluginInterface* plugin) {
        registry()[type] = plugin;
    }
    
    // 创建插件
    static PluginInterface* createPlugin(const QString& type) {
        if (registry().contains(type)) {
            return registry()[type];
        }
        return nullptr;
    }
};

// 使用
void usePlugins() {
    // 注册
    PluginFactory::registerPlugin("image", new ImagePlugin);
    PluginFactory::registerPlugin("audio", new AudioPlugin);
    
    // 创建并使用
    PluginInterface* plugin = PluginFactory::createPlugin("image");
    if (plugin) {
        qDebug() << "插件:" << plugin->name();
        plugin->execute();
    }
}
```

## 十一、常见面试题与解答

### 1. 单例模式的实现方式有哪些？

**答：**
- 饿汉式：程序启动时创建，线程安全但浪费资源
- 懒汉式：第一次使用时创建，需要双重检查锁定
- C++11局部静态：最简单，线程安全且延迟初始化

### 2. 工厂模式和抽象工厂的区别？

**答：**
- 工厂模式：一个工厂创建一种产品
- 抽象工厂：一个工厂创建一系列相关产品

### 3. MVC和MVVM的区别？

**答：**
- MVC：Controller处理用户输入，更新Model和View
- MVVM：ViewModel双向绑定Model和View，不需要手动更新

### 4. 装饰模式和继承的区别？

**答：**
- 继承：编译时确定，静态的，会导致类爆炸
- 装饰：运行时动态添加，灵活，可以组合多个装饰器

### 5. 建造者模式和工厂模式的区别？

**答：**
- 工厂模式：直接创建完整对象
- 建造者模式：分步骤构建对象，可以控制构建过程

### 6. 你在项目中用过哪些设计模式？

**答：**
（根据个人经验回答，例如：）
- 单例：配置管理器、日志系统
- 工厂：插件系统、不同格式的文件处理器
- MVC：数据表格显示
- 观察者：信号槽机制
