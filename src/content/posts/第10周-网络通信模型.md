## 一、同步 vs 异步

### 1.1 同步通信

**特点：** 发送方发送请求后，必须等待接收方响应才能继续。

**比喻：** 打电话
- 你打电话给别人，必须等对方接听才能说话
- 对方不接，你就一直等着

```
客户端                    服务器
  │                        │
  │──── 请求 ────────────→│
  │                        │
  │  (等待...)             │
  │                        │
  │←─── 响应 ─────────────│
  │                        │
  │  继续执行              │
```

**代码示例：**
```cpp
// 同步：recv会阻塞，直到有数据
char buffer[1024];
int n = recv(socket, buffer, sizeof(buffer), 0);  // 卡在这里等待
// 只有收到数据后才会执行下面
std::cout << "收到数据" << std::endl;
```

### 1.2 异步通信

**特点：** 发送方发送请求后，不等待响应，继续做其他事情。

**比喻：** 发短信
- 你发短信给别人，不用等回复，可以继续做其他事
- 对方回复了，你会收到通知

```
客户端                    服务器
  │                        │
  │──── 请求 ────────────→│
  │  继续做其他事          │
  │                        │
  │←─── 响应 ─────────────│
  │  (收到通知)            │
```

### 1.3 阻塞 vs 非阻塞

| 模式 | 特点 | 比喻 |
|------|------|------|
| 阻塞 | 操作未完成就等待 | 排队等叫号 |
| 非阻塞 | 操作未完成立即返回 | 拿号去逛街，叫号了再来 |

```cpp
// 设置Socket为非阻塞
int flags = fcntl(socket, F_GETFL, 0);
fcntl(socket, F_SETFL, flags | O_NONBLOCK);

// 非阻塞recv：没有数据立即返回-1
int n = recv(socket, buffer, sizeof(buffer), 0);
if (n == -1 && errno == EAGAIN) {
    // 没有数据，但不阻塞，可以做其他事
}
```

## 二、I/O多路复用

### 2.1 为什么需要I/O多路复用？

**问题场景：** 服务器要同时处理1000个客户端连接

**方案1：多线程（不好）**
```
每个连接一个线程 → 1000个线程 → 资源消耗大，切换开销大
```

**方案2：I/O多路复用（好）**
```
一个线程监控所有连接 → 哪个有数据就处理哪个 → 高效
```

### 2.2 三种I/O多路复用方式

| 方式 | 出现时间 | 最大连接数 | 效率 |
|------|---------|-----------|------|
| select | 最早 | 1024 | 低 |
| poll | 较晚 | 无限制 | 中 |
| epoll | 最新(Linux) | 无限制 | 高 |

## 三、select

### 3.1 select原理

select用一个集合来监控多个文件描述符（Socket）：

```
监控集合: {socket1, socket2, socket3, ...}
                ↓
         select等待
                ↓
    哪个Socket有数据就返回哪个
```

### 3.2 select代码示例

```cpp
#include <iostream>
#include <cstring>
#include <unistd.h>
#include <arpa/inet.h>
#include <sys/socket.h>
#include <sys/select.h>

#define MAX_CLIENTS 1024

int main() {
    // 创建服务器Socket
    int serverSocket = socket(AF_INET, SOCK_STREAM, 0);
    
    sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_addr.s_addr = INADDR_ANY;
    serverAddr.sin_port = htons(8888);
    
    bind(serverSocket, (sockaddr*)&serverAddr, sizeof(serverAddr));
    listen(serverSocket, 5);
    
    // fd_set是select使用的集合
    fd_set readSet;
    int maxFd = serverSocket;
    
    std::cout << "select服务器启动..." << std::endl;
    
    while (true) {
        // 每次循环都要重新设置集合
        FD_ZERO(&readSet);           // 清空集合
        FD_SET(serverSocket, &readSet);  // 加入服务器Socket
        
        // 把所有客户端Socket加入集合
        // (这里简化处理，实际应该维护一个客户端列表)
        
        // select等待，timeout设为NULL表示无限等待
        int readyCount = select(maxFd + 1, &readSet, nullptr, nullptr, nullptr);
        
        if (readyCount < 0) {
            std::cerr << "select错误" << std::endl;
            break;
        }
        
        // 检查服务器Socket是否有新连接
        if (FD_ISSET(serverSocket, &readSet)) {
            sockaddr_in clientAddr;
            socklen_t clientLen = sizeof(clientAddr);
            int clientSocket = accept(serverSocket, (sockaddr*)&clientAddr, &clientLen);
            
            std::cout << "新客户端连接: " << clientSocket << std::endl;
            maxFd = std::max(maxFd, clientSocket);
        }
        
        // 检查各个客户端是否有数据
        // (这里简化处理)
    }
    
    close(serverSocket);
    return 0;
}
```

### 3.3 select的缺点

1. **最大连接数限制：** 默认1024（FD_SETSIZE）
2. **效率低：** 每次都要遍历所有Socket
3. **需要重新设置集合：** 每次调用前都要FD_ZERO和FD_SET

## 四、poll

### 4.1 poll原理

poll用数组代替select的集合，没有连接数限制。

```cpp
struct pollfd {
    int fd;      // 文件描述符
    short events; // 关注的事件
    short revents; // 实际发生的事件
};
```

### 4.2 poll代码示例

```cpp
#include <iostream>
#include <cstring>
#include <unistd.h>
#include <arpa/inet.h>
#include <sys/socket.h>
#include <poll.h>

#define MAX_EVENTS 1024

int main() {
    int serverSocket = socket(AF_INET, SOCK_STREAM, 0);
    
    sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_addr.s_addr = INADDR_ANY;
    serverAddr.sin_port = htons(8888);
    
    bind(serverSocket, (sockaddr*)&serverAddr, sizeof(serverAddr));
    listen(serverSocket, 5);
    
    // pollfd数组
    struct pollfd fds[MAX_EVENTS];
    memset(fds, 0, sizeof(fds));
    
    fds[0].fd = serverSocket;
    fds[0].events = POLLIN;  // 关注可读事件
    
    int nfds = 1;  // 当前监控的数量
    
    std::cout << "poll服务器启动..." << std::endl;
    
    while (true) {
        // poll等待
        int readyCount = poll(fds, nfds, -1);  // -1表示无限等待
        
        if (readyCount < 0) {
            std::cerr << "poll错误" << std::endl;
            break;
        }
        
        // 检查服务器Socket
        if (fds[0].revents & POLLIN) {
            sockaddr_in clientAddr;
            socklen_t clientLen = sizeof(clientAddr);
            int clientSocket = accept(serverSocket, (sockaddr*)&clientAddr, &clientLen);
            
            if (nfds < MAX_EVENTS) {
                fds[nfds].fd = clientSocket;
                fds[nfds].events = POLLIN;
                nfds++;
                
                std::cout << "新客户端: " << clientSocket << std::endl;
            }
        }
        
        // 检查客户端
        for (int i = 1; i < nfds; i++) {
            if (fds[i].revents & POLLIN) {
                char buffer[1024];
                int n = recv(fds[i].fd, buffer, sizeof(buffer), 0);
                
                if (n <= 0) {
                    // 客户端断开
                    close(fds[i].fd);
                    fds[i] = fds[nfds - 1];  // 用最后一个替换
                    nfds--;
                    i--;
                } else {
                    buffer[n] = '\0';
                    std::cout << "收到: " << buffer << std::endl;
                    send(fds[i].fd, buffer, n, 0);  // 回显
                }
            }
        }
    }
    
    close(serverSocket);
    return 0;
}
```

### 4.3 poll vs select

| 特性 | select | poll |
|------|--------|------|
| 最大连接数 | 1024 | 无限制 |
| 数据结构 | fd_set位图 | pollfd数组 |
| 效率 | 低（遍历所有） | 低（遍历所有） |
| 可移植性 | 好 | 较好 |

## 五、epoll（重点！）

### 5.1 epoll原理

epoll是Linux特有的高效I/O多路复用机制。

**核心思想：** 只返回有事件的Socket，不需要遍历所有

```
select/poll: 遍历1000个Socket → 找到3个有事件的
epoll:       直接返回3个有事件的Socket
```

### 5.2 epoll三种工作模式

```
epoll_create()  →  创建epoll实例
epoll_ctl()     →  添加/删除/修改监控的Socket
epoll_wait()    →  等待事件发生
```

### 5.3 epoll代码示例（ET模式）

```cpp
#include <iostream>
#include <cstring>
#include <unistd.h>
#include <arpa/inet.h>
#include <sys/socket.h>
#include <sys/epoll.h>
#include <fcntl.h>

#define MAX_EVENTS 1024
#define BUFFER_SIZE 1024

// 设置非阻塞
void setNonBlocking(int fd) {
    int flags = fcntl(fd, F_GETFL, 0);
    fcntl(fd, F_SETFL, flags | O_NONBLOCK);
}

int main() {
    // 创建服务器Socket
    int serverSocket = socket(AF_INET, SOCK_STREAM, 0);
    setNonBlocking(serverSocket);
    
    int opt = 1;
    setsockopt(serverSocket, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
    
    sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_addr.s_addr = INADDR_ANY;
    serverAddr.sin_port = htons(8888);
    
    bind(serverSocket, (sockaddr*)&serverAddr, sizeof(serverAddr));
    listen(serverSocket, 5);
    
    // 1. 创建epoll实例
    int epollFd = epoll_create1(0);
    
    // 2. 添加服务器Socket到epoll
    struct epoll_event event;
    event.events = EPOLLIN;  // 监控可读事件
    event.data.fd = serverSocket;
    epoll_ctl(epollFd, EPOLL_CTL_ADD, serverSocket, &event);
    
    struct epoll_event events[MAX_EVENTS];
    
    std::cout << "epoll服务器启动..." << std::endl;
    
    while (true) {
        // 3. 等待事件
        int readyCount = epoll_wait(epollFd, events, MAX_EVENTS, -1);
        
        for (int i = 0; i < readyCount; i++) {
            if (events[i].data.fd == serverSocket) {
                // 新连接
                while (true) {
                    sockaddr_in clientAddr;
                    socklen_t clientLen = sizeof(clientAddr);
                    int clientSocket = accept(serverSocket, (sockaddr*)&clientAddr, &clientLen);
                    
                    if (clientSocket < 0) break;
                    
                    setNonBlocking(clientSocket);
                    
                    // 添加到epoll（ET模式）
                    event.events = EPOLLIN | EPOLLET;  // 边缘触发
                    event.data.fd = clientSocket;
                    epoll_ctl(epollFd, EPOLL_CTL_ADD, clientSocket, &event);
                    
                    std::cout << "新客户端: " << clientSocket << std::endl;
                }
            } else {
                // 客户端数据
                int clientSocket = events[i].data.fd;
                char buffer[BUFFER_SIZE];
                
                while (true) {
                    int n = recv(clientSocket, buffer, BUFFER_SIZE, 0);
                    
                    if (n < 0) {
                        if (errno == EAGAIN || errno == EWOULDBLOCK) {
                            // 数据读完
                            break;
                        }
                        // 错误
                        goto remove_client;
                    } else if (n == 0) {
                        // 客户端断开
                        goto remove_client;
                    } else {
                        buffer[n] = '\0';
                        std::cout << "收到[" << clientSocket << "]: " << buffer << std::endl;
                        send(clientSocket, buffer, n, 0);  // 回显
                    }
                }
                
                continue;
                
            remove_client:
                std::cout << "客户端断开: " << clientSocket << std::endl;
                close(clientSocket);
                epoll_ctl(epollFd, EPOLL_CTL_DEL, clientSocket, nullptr);
            }
        }
    }
    
    close(epollFd);
    close(serverSocket);
    return 0;
}
```

## 六、边缘触发（ET）vs 水平触发（LT）

### 6.1 概念对比

**水平触发（Level Triggered, LT）：**
- 只要Socket缓冲区有数据，就会一直通知
- 默认模式，select/poll都是LT

**边缘触发（Edge Triggered, ET）：**
- 只在状态变化时通知一次
- 必须一次把数据读完
- 效率更高，但编程更复杂

### 6.2 比喻理解

**LT（水平触发）：** 像门铃一直响
- 有人按门铃，铃声一直响
- 你什么时候开门都可以

**ET（边缘触发）：** 像门铃只响一次
- 有人按门铃，只响一声
- 你必须立即去开门，否则就错过了

### 6.3 代码对比

```cpp
// LT模式（默认）
event.events = EPOLLIN;  // 水平触发
// 可以分多次读取数据

// ET模式
event.events = EPOLLIN | EPOLLET;  // 边缘触发
// 必须一次读完所有数据，配合非阻塞Socket
```

### 6.4 ET模式注意事项

```cpp
// ET模式必须：
// 1. 设置非阻塞Socket
setNonBlocking(fd);

// 2. 循环读取直到EAGAIN
while (true) {
    int n = recv(fd, buffer, size, 0);
    if (n < 0 && errno == EAGAIN) {
        break;  // 数据读完
    }
    // 处理数据...
}
```

## 七、Reactor模式

### 7.1 什么是Reactor模式？

Reactor是一种事件驱动的网络编程模式，epoll通常配合Reactor使用。

```
                    ┌─────────────┐
    客户端请求 ───→│   Reactor   │
                    │  (epoll)    │
                    └──────┬──────┘
                           │ 分发事件
              ┌────────────┼────────────
              ↓            ↓            ↓
        ┌─────────┐  ┌─────────┐  ┌─────────┐
        │ Handler │  │ Handler │  │ Handler │
        │   A     │  │   B     │  │   C     │
        └─────────┘  └─────────┘  └─────────┘
```

### 7.2 简易Reactor实现

```cpp
#include <iostream>
#include <cstring>
#include <unistd.h>
#include <arpa/inet.h>
#include <sys/socket.h>
#include <sys/epoll.h>
#include <fcntl.h>
#include <functional>
#include <map>

class Reactor {
public:
    Reactor() {
        epollFd = epoll_create1(0);
    }
    
    // 注册事件处理函数
    void addEvent(int fd, std::function<void(int)> handler) {
        struct epoll_event event;
        event.events = EPOLLIN | EPOLLET;
        event.data.fd = fd;
        epoll_ctl(epollFd, EPOLL_CTL_ADD, fd, &event);
        handlers[fd] = handler;
    }
    
    // 移除事件
    void removeEvent(int fd) {
        epoll_ctl(epollFd, EPOLL_CTL_DEL, fd, nullptr);
        handlers.erase(fd);
        close(fd);
    }
    
    // 运行事件循环
    void run() {
        struct epoll_event events[1024];
        
        while (true) {
            int n = epoll_wait(epollFd, events, 1024, -1);
            
            for (int i = 0; i < n; i++) {
                int fd = events[i].data.fd;
                if (handlers.count(fd)) {
                    handlers[fd](fd);  // 调用处理函数
                }
            }
        }
    }
    
private:
    int epollFd;
    std::map<int, std::function<void(int)>> handlers;
};

// 使用示例
int main() {
    Reactor reactor;
    
    // 创建服务器Socket
    int serverSocket = socket(AF_INET, SOCK_STREAM, 0);
    
    sockaddr_in addr;
    addr.sin_family = AF_INET;
    addr.sin_addr.s_addr = INADDR_ANY;
    addr.sin_port = htons(8888);
    
    bind(serverSocket, (sockaddr*)&addr, sizeof(addr));
    listen(serverSocket, 5);
    
    // 注册接受连接的处理函数
    reactor.addEvent(serverSocket, [&reactor, serverSocket](int fd) {
        sockaddr_in clientAddr;
        socklen_t len = sizeof(clientAddr);
        int clientSocket = accept(fd, (sockaddr*)&clientAddr, &len);
        
        // 注册客户端数据处理
        reactor.addEvent(clientSocket, [&reactor](int clientFd) {
            char buffer[1024];
            int n = recv(clientFd, buffer, sizeof(buffer), 0);
            
            if (n <= 0) {
                reactor.removeEvent(clientFd);
                return;
            }
            
            buffer[n] = '\0';
            std::cout << "收到: " << buffer << std::endl;
            send(clientFd, buffer, n, 0);
        });
    });
    
    std::cout << "Reactor服务器启动..." << std::endl;
    reactor.run();
    
    return 0;
}
```

## 八、Linux I/O多路复用 → Qt网络通信（重点！）

### 8.1 Qt如何实现高效的网络通信

Qt内部在不同平台使用不同的I/O多路复用机制：
- Linux: epoll
- macOS: kqueue  
- Windows: IOCP

Qt开发者不需要直接使用epoll，Qt的**事件循环+信号槽**已经封装好了！

### 8.2 Qt事件循环 = Reactor模式

Qt的事件循环本质上就是Reactor模式：

```cpp
// Qt事件循环伪代码
int QCoreApplication::exec() {
    while (!exitRequested) {
        // 1. 等待事件（内部使用epoll/select等）
        QEvent event = waitForEvent();
        
        // 2. 分发事件
        switch (event.type()) {
            case SocketRead:
                emit socket->readyRead();  // 触发信号
                break;
            case SocketWrite:
                emit socket->bytesWritten();
                break;
            case Timer:
                emit timer->timeout();
                break;
            // ... 其他事件
        }
    }
    return 0;
}
```

### 8.3 Qt TCP服务器（内部使用epoll）

```cpp
#include <QTcpServer>
#include <QTcpSocket>
#include <QDebug>

// 这个简单例子内部就使用了epoll！
class SimpleServer {
private:
    QTcpServer* server;
    
public:
    SimpleServer() {
        server = new QTcpServer;
        
        // 监听端口 - 内部创建socket + bind + listen + 加入epoll
        server->listen(QHostAddress::Any, 8888);
        
        // 注册事件处理函数 - 内部加入epoll的事件回调
        connect(server, &QTcpServer::newConnection, []() {
            QTcpSocket* client = server->nextPendingConnection();
            qDebug() << "新连接:" << client->peerAddress();
            
            // 客户端可读事件 - 内部监听客户端socket的EPOLLIN
            connect(client, &QTcpSocket::readyRead, [client]() {
                QByteArray data = client->readAll();
                qDebug() << "收到数据:" << data;
                client->write("Echo: " + data);
            });
            
            // 客户端断开事件
            connect(client, &QTcpSocket::disconnected, [client]() {
                qDebug() << "客户端断开";
                client->deleteLater();
            });
        });
    }
    
    ~SimpleServer() {
        server->close();
    }
};

// 使用
int main(int argc, char* argv[]) {
    QCoreApplication app(argc, argv);
    
    SimpleServer server;
    
    // 启动事件循环 - 内部调用epoll_wait循环
    return app.exec();
}
```

### 8.4 Qt vs 原生epoll 对比

| 特性 | 原生epoll | Qt |
|------|----------|-----|
| 代码复杂度 | 高（需要手动管理） | 低（信号槽自动管理） |
| 事件类型 | 只有I/O事件 | I/O、定时器、GUI等 |
| 跨平台 | 仅Linux | 跨平台 |
| 与GUI集成 | 无 | 完美集成 |
| 性能 | 最高 | 略低但足够 |
| 推荐场景 | 高性能服务器 | GUI应用、嵌入式 |

### 8.5 Qt定时器（内部也使用epoll）

```cpp
#include <QTimer>

class TimerExample {
```

### 8.6 Qt处理大量并发连接

虽然Qt使用了epoll，但仍需注意性能优化：

```cpp
#include <QTcpServer>
#include <QTcpSocket>
#include <QVector>

class HighPerformanceServer {
private:
    QTcpServer* server;
    QVector<QTcpSocket*> clients;
    
public:
    HighPerformanceServer() {
        server = new QTcpServer;
        
        // 优化1：使用LowDelay选项
        server->setSocketOptions(QAbstractSocket::LowDelayOption);
        
        // 优化2：设置较大的监听队列
        if (server->listen(QHostAddress::Any, 8888)) {
            qDebug() << "服务器启动";
        }
        
        connect(server, &QTcpServer::newConnection, this, &HighPerformanceServer::newClient);
    }
    
    void newClient() {
        QTcpSocket* client = server->nextPendingConnection();
        clients.append(client);
        
        // 优化3：使用DirectConnection减少信号槽开销
        connect(client, &QTcpSocket::readyRead, this, &HighPerformanceServer::handleData,
                Qt::DirectConnection);
        
        connect(client, &QTcpSocket::disconnected, this, &HighPerformanceServer::clientLeft);
    }
    
    void handleData() {
        QTcpSocket* client = qobject_cast<QTcpSocket*>(sender());
        if (!client) return;
        
        // 优化4：批量读取
        QByteArray data = client->readAll();
        // 处理数据...
    }
    
    void clientLeft() {
        QTcpSocket* socket = qobject_cast<QTcpSocket*>(sender());
        clients.removeAll(socket);
        socket->deleteLater();
    }
};
```

### 8.7 异步编程在Qt中的实践

Qt的异步编程模型让你不用关心底层细节：

#### 同步 vs 异步 在Qt中的体现

```cpp
// 同步方式（阻塞）- 不推荐在GUI线程使用
QTcpSocket socket;
socket.connectToHost("server.com", 8888);
socket.waitForConnected();  // 阻塞等待
socket.write("Hello");
socket.waitForBytesWritten();  // 阻塞等待

// 异步方式（非阻塞）- 推荐
QTcpSocket* socket = new QTcpSocket;
socket->connectToHost("server.com", 8888);

// 使用信号槽，不阻塞
connect(socket, &QTcpSocket::connected, []() {
    qDebug() << "连接成功";
    socket->write("Hello");
});

connect(socket, &QTcpSocket::bytesWritten, [](qint64 bytes) {
    qDebug() << "发送完成:" << bytes << "字节";
});
```

## 十二、常见面试题与解答

### 1. select、poll、epoll的区别？

**答：**
- select：最早出现，最大1024连接，每次遍历所有fd，效率低
- poll：无连接数限制，仍然遍历所有fd，效率中等
- epoll：Linux特有，只返回有事件的fd，效率最高

### 2. 什么是I/O多路复用？

**答：**
I/O多路复用允许一个线程同时监控多个I/O操作（如Socket），当某个I/O就绪时通知程序处理，避免为每个连接创建一个线程。

### 3. ET和LT模式的区别？

**答：**
- LT（水平触发）：只要缓冲区有数据就一直通知，默认模式
- ET（边缘触发）：只在状态变化时通知一次，必须一次读完所有数据

### 4. epoll为什么比select高效？

**答：**
- select：每次调用都要遍历所有fd，时间复杂度O(n)
- epoll：使用红黑树管理fd，只有事件发生时才返回，时间复杂度O(1)

### 5. 什么是Reactor模式？

**答：**
Reactor是一种事件驱动的编程模式，核心思想：
1. 将事件注册到Reactor（epoll）
2. Reactor等待事件发生
3. 事件发生时调用对应的Handler处理

### 6. 惊群效应是什么？如何解决？

**答：**
惊群效应：多个进程/线程等待同一个事件，事件发生时全部被唤醒但只有一个能处理，其他继续睡眠。
解决方法：
- 使用EPOLLEXCLUSIVE标志（Linux 4.5+）
- 使用单线程Reactor+多线程Worker

### 7. Qt中如何处理大量网络请求？

**答：**
- 使用QNetworkAccessManager的异步API
- 限制并发请求数量
- 使用连接池复用连接
- 使用缓存减少请求

### 8. 同步和异步在Qt中的体现？

**答：**
- 同步：waitForConnected()、waitForReadyRead()等（阻塞）
- 异步：信号槽机制（非阻塞，推荐）
